§01 — BRAND

Trading Desk

A professional, market-standard aesthetic: dark canvas, electric-blue conviction, and the green/red P&L language every trader already reads at a glance. Built to feel like a tool a serious user would trust with real capital.

Personality

Pacekeeper is a steady-handed operator, not a trading guru. It doesn't promise wins; it promises discipline. It shows its work in plain language, never asks the user to read a log file, and confirms anything that touches real money.

If it had a voice, it would be a librarian who used to be a risk officer.

Pillars
  1. Composure — calm typography, tabular numerals, generous air.
  2. Legibility — plain language first, jargon hidden behind disclosure.
  3. Reversibility — every dangerous action is two clicks and a name.
  4. Provenance — every number is footnoted with where it came from.
We are
Patient · Documentary · Disciplined · Quiet · Warm
We are not
Hype · Neon · Gambling · "Crushing it" · Loud

§05 — ICONOGRAPHY · MOTION

Stroke icons drawn at 1.25px on a 16-grid; never filled, never multicolour. Motion is paper-fast — short eases for state, longer eases for context.

Rules
  • Numerals always tabular, mono, weight 500.
  • Y-axis starts at zero only when context demands it; otherwise zoom but annotate.
  • One conviction colour per chart — green/red on PnL only.
  • No 3D, no rounded bars, no drop shadows, no animated reveals.
  • Annotate, don't decorate: every spike gets a date or event.
  • Sparkline ≤ 32px tall in cards, ≤ 16px inline.
  • Risk gauges read 0–10 in plain language: Calm · Steady · Warm · Hot.
  • Always include the source and timestamp in caption.

§09 — JOURNEYS

Eight flows, mapped

Each journey is short, guided, and reversible. Anything that touches real money requires a typed confirmation; anything that costs money requires a second screen.

A · First launch
  1. Installer opens a single welcome window.
  2. License key entered or "Start trial" — 14 days, no card.
  3. Pick what to trade: Stocks · Prediction · Both.
  4. Connect an account — keys are validated live.
  5. Set safety dials: daily loss · max positions · cooldown.
  6. Demo run — paper trade for 24h before Live unlocks.
B · Activation
  1. Paste license key (or sign in to recover).
  2. Online check + offline fallback (signed token).
  3. Bound to this machine name; show 2 of 2 seats used.
  4. "Trial → paid" upgrades in place; no reinstall.
C · Connect a market
  1. Pick the market provider.
  2. Step-by-step: open broker site → API panel → copy key.
  3. Paste; we test connection live (green check or plain error).
  4. Choose Demo or Live; default Demo.
D · Demo → Live
  1. Live tab is locked until 24h Demo completes.
  2. Show Demo report: trades, win-rate, max drawdown.
  3. "Switch to Live" requires typing the word LIVE.
  4. Mode bar turns amber; subtle screen tint shifts.
  5. "Switch back to Demo" is one click, anytime.
E · Daily use
  1. Open Pacekeeper → Today screen.
  2. Read three numbers: P&L · positions · headroom.
  3. Skim the activity feed in plain English.
  4. Pause at any time with ⌘. — resumes on next session.
F · Update
  1. Background check on launch; never blocks trading.
  2. Notice in top right: "Update available · what's new".
  3. One-click install during a Pause window.
  4. Failed update rolls back automatically; banner explains why.
G · Error recovery
  1. Broker unreachable → auto-pause, retry every 30s.
  2. Plain-language banner with last-success timestamp.
  3. "Copy diagnostic bundle" instead of asking for logs.
  4. Resume requires no action once link is restored.
H · Emergency stop
  1. Big Pause in the title bar pauses new trades.
  2. Stop everything in safety panel closes all positions at market, with confirmation.
  3. Logged with reason and timestamp; stays visible for 24h.
§10 — MODES

One app, two postures

Beginner mode is the default. It hides everything that isn't a decision the user cares about. Advanced exposes the same data with more density, more knobs, and live numerals.

Beginner
  • Three KPIs only: P&L today · positions · headroom.
  • Activity in sentences, not rows.
  • Sliders for safety, not number inputs.
  • Confirm dialogs name the consequence: "This sells 8 shares of AAPL at market".
  • Tooltips written for someone who has never opened a brokerage account.
Advanced
  • Six-up KPI strip with sparklines.
  • Tabular activity with column controls.
  • Number inputs with units; ⌘-K command palette.
  • Strategy JSON viewer; raw signal log.
  • Per-market overrides; multi-account exposure view.

My broker says "Unauthorized"

This usually means the API key is correct but the broker hasn't enabled your account for API trading yet. Walk through the three steps below — most people fix this in under a minute.

§25 — TRUST

Trust as a series of small, kept promises

Trust isn't a badge. It's whether the app does exactly what it said it would do. Pacekeeper signals competence with restraint, never with reassurance theatre.

DATA

Local first

Keys live in the OS keychain. Trade data stays on disk, encrypted at rest. We don't have a server that knows what you trade.

DIAGNOSTICS

Anonymous & opt-in

Crash reports and anonymous usage stats are off by default. The settings screen explains exactly what's sent — no marketing copy.

CODE

Notarised & signed

Apple Notarised, Microsoft signed. Every binary's SHA-256 lives on the changelog page so you can verify it.

REVERSIBILITY

Pause is global

⌘. anywhere pauses trading. Stop closes all positions with a typed confirmation. There is always a way out.

DEMO

Required, not suggested

Live unlocks only after a 24-hour Demo run. Switching back to Demo is one click — and we don't hide it.

REFUND

14 days, no questions

Cancel from inside the app. Refund within 14 days from a single email. No retention call.

§28 — ACCESS

Accessibility checklist

Numerals are tabular. Money never leans on colour alone. Every flow is reachable on the keyboard.

VISUAL
  • WCAG AA minimum, AAA for confirmations and money figures.
  • Up/down conveyed by sign + colour + weight, never colour alone.
  • Dark theme parity tested at the same contrasts.
  • Focus rings 3px, accent-soft, never removed on click.
  • Honours prefers-reduced-motion — disables y-translate, keeps opacity.
INPUT
  • Every action reachable by keyboard. ⌘. pause, ⌘? help, ⌘K palette.
  • Tab order matches reading order. Skip-to-content link in the shell.
  • Form errors tied to fields with aria-describedby.
  • Modals trap focus, restore on close, can be dismissed with Esc.
  • Live regions for activity feed updates (polite, throttled).
